# Internacionalizacion y Theming — Starter App

> next-intl, dark/light mode y emails traducidos.

---

## next-intl — Configuracion

Starter App soporta **espanol (es)**, **ingles (en)** y **catalan (ca)**.

### Estructura de Archivos

```
i18n/
├── routing.ts      # Definicion de rutas y locales
├── request.ts      # Carga de mensajes segun locale
└── navigation.ts   # Link, useRouter, etc. localizados

messages/
├── es.json         # Traducciones espanol (default)
├── en.json         # Traducciones ingles
├── ca.json         # Traducciones catalan
└── es.d.json.ts    # TypeScript declarations (AOT precompilation)
```

### Routing (`i18n/routing.ts`)

```typescript
import { defineRouting } from 'next-intl/routing'

export const routing = defineRouting({
  locales: ['es', 'en', 'ca'],
  defaultLocale: 'es',
  localePrefix: 'as-needed', // Solo /en/* y /ca/* tienen prefijo
  localeDetection: false, // Control manual
})
```

## Uso en Componentes

### Server Components

```tsx
import { getTranslations } from 'next-intl/server'

export default async function Page() {
  const t = await getTranslations('landing')
  return <h1>{t('hero.title')}</h1>
}
```

### Client Components

```tsx
'use client'
import { useTranslations } from 'next-intl'

export function MyComponent() {
  const t = useTranslations('common')
  return <button>{t('submit')}</button>
}
```

## Language Switcher

```tsx
'use client'
import { useLocale } from 'next-intl'
import { usePathname, useRouter } from '@/i18n/navigation'

export function LanguageSwitcher() {
  const locale = useLocale()
  const pathname = usePathname()
  const router = useRouter()

  const switchLocale = (newLocale: 'es' | 'en' | 'ca') => {
    router.replace(pathname, { locale: newLocale })
  }
  // ...
}
```

## Emails Internacionalizados

```tsx
// emails/i18n.tsx
import esMessages from '@/messages/es.json'
import enMessages from '@/messages/en.json'

export type EmailLocale = 'es' | 'en' | 'ca'

export function getEmailTranslations(locale: EmailLocale = 'es') {
  const messages = { es: esMessages.emails, en: enMessages.emails }
  return messages[locale] ?? messages.es
}
```

## Reglas i18n

- SIEMPRE usar `Link` de `@/i18n/navigation`, NUNCA `next/link`
- SIEMPRE usar `useRouter` de `@/i18n/navigation`, NUNCA `next/navigation`
- El locale default (`es`) NO tiene prefijo en la URL
- Nuevos textos: agregar en `es.json`, `en.json` Y `ca.json`
- Emails se traducen automaticamente segun locale del usuario
- AOT precompilation habilitada via `createMessagesDeclaration`

## Theming — Dark/Light Mode

### ThemeProvider (`app/[locale]/layout.tsx`)

```tsx
import { ThemeProvider } from 'next-themes'

;<html suppressHydrationWarning>
  <body>
    <ThemeProvider
      attribute="class"
      defaultTheme="system"
      enableSystem
      disableTransitionOnChange
    >
      {children}
    </ThemeProvider>
  </body>
</html>
```

### Reglas de Theming

- SIEMPRE usar `suppressHydrationWarning` en `<html>`
- NUNCA acceder a `theme` durante SSR
- CSS variables definen colores — Tailwind los consume
- `disableTransitionOnChange` evita flash al cambiar
